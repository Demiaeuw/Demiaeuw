-Installer WSL2

	-Dans PowerShell
		wsl --install
	-Vérifier la version
		wsl -l -v

-Installer Ubuntu

	-Dans PowerShell
		wsl --list --online
	-Installer une distro spécifique
		wsl --install -d <Nom_de_la_Distribution>

-Mise a jour WSL2

	-Dans PowerShell
		wsl --update

-Redemarrer WSL2

	-Dans PowerShell
		wsl --shutdown

-Redemarrer le pc quand il le demande

-Utilisé VScode

	-Dans VScode
		-Installer l'addon Explorateur Distant
		-Connectez vous a votre distro Ubuntu via cet explorateur Distant

			Cela permet de lancer VScode directement dans l'environement Linux de WSL 
			pour eviter les problemes. Si vous ouvrez le dossier via l'explorateur Windows directement 
			cela peut generer des erreur de compilation et autre.

			Il faudra sync vos addon preferer qui sont sur le vscode de base dans Ubuntu. pour ca dans Addons vous avez
			une nouvelle cathegorie : WSL : <votre distribution> avec un nuage avec un fleche, cliquez dessus et
			séléctionnez les addons que vous voulez utilisé.

			Pour quitter WSL dans VScode cliquez en bas a gauche sur l'icone a double chevron pour acceder au menu
			des connection a distance.

-Accés a WSL via terminal

	-Je vous conseil de passer par l'apps terminal de windows et d'ouvrir un nouvel onglet en selectionant Ubuntu
	Si vous tapez wsl dans powershell vous allez arriver dans windows avec wsl. Ouvrir un nouvel onglet sous Ubuntu 
	vous permet d'etre directement dans WSL mais surtout de ne pas en sortir et donc eviter tout probleme de sync
	ou autre problemes de compatibilité
	-Partez du principe que WSL est une machine virtuel (meme si ce n'est pas le cas) dans un premier temps.
